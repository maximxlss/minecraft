# mc.xls.msk.ru

_[English version](README.md)_

Minecraft-сервер, который может строить кто угодно. Здесь нет админа,
который вручную что-то настраивает за консолью — вообще всё (плагины,
настройки, даже список админов) хранится в этом репозитории, и именно
слияние pull request'а меняет то, что реально работает на сервере.

## Играть

- **Адрес:** `mc.xls.msk.ru`, порт по умолчанию (`25565`) — просто
  вставьте адрес, порт указывать не нужно.
- **Версия:** сейчас Paper **26.1.2**, но сервер всегда следует за самой
  свежей стабильной сборкой Paper (см. `VERSION: LATEST` в
  `docker-compose.yml`), так что со временем версия будет меняться. Держать
  версию клиента строго в ногу с сервером не обязательно — специально для
  этого установлены [ViaVersion](https://modrinth.com/plugin/viaversion) и
  [ViaBackwards](https://modrinth.com/plugin/viabackwards), чтобы чуть более старые или новые клиенты всё равно могли подключиться.
- **Без whitelist, без аккаунта Mojang.** При первом заходе выполните
  `/register <пароль> <пароль>`, при каждом следующем — `/login <пароль>`.
  Выбирайте пароль, который не жалко использовать для Minecraft-сервера, а
  не тот, что используется где-то действительно важном — аккаунты здесь
  локальные, к Mojang никак не привязаны.
- **Бэкапы делаются автоматически** каждые 24 часа, так что гриферство или
  неудачная стройка — это поправимо, а не катастрофа.

## Участвовать

Хотите добавить плагин, поменять настройку или предложить кого-то в
админы? Откройте pull request. После ревью и слияния изменения появятся
на реальном сервере в течение минуты-двух — это и есть весь процесс
деплоя. Как это сделать — в
[`docs/CONTRIBUTING.md`](docs/CONTRIBUTING.md) (на английском); полное
описание архитектуры и логики решений — в [`SPEC.md`](SPEC.md).

## Обслуживание сервера

### Первоначальная настройка хоста (разово, вручную)

```
ssh minecraft
git clone <this repo> && cd minecraft-server
./provisioning/bootstrap.sh
```

Затем скопируйте `docker-compose.yml` (и `plugin-configs/`, если там
что-то есть) в `/opt/minecraft/` и запустите `docker compose up -d` уже
оттуда. После этого при каждом мерже в `main` дальше работает `deploy.yml`.

### Настройка CI

В GitHub-репозитории для `deploy.yml` нужны:

- Secret `DEPLOY_SSH_KEY` — приватная половина deploy-ключа, публичная
  половина которого лежит в `~/.ssh/authorized_keys` на хосте.
- Переменные `DEPLOY_HOST` и `DEPLOY_USER` — адрес хоста и SSH-пользователь
  для подключения.

Также стоит настроить branch protection для `main`
(`Settings → Branches → Add branch ruleset`): обязательные проверки
`Validate / lint` и `Validate / smoke-test`, плюс требование, чтобы ветка
была актуальной перед мержем. GitHub позволяет выбрать проверку как
обязательную только после того, как она хотя бы раз отработала — то есть
это делается уже после первого PR.
